PCSX Script Opcode Extract / Reimport Tool

Loads packed .PCSX files, resolves hash names with string_hash_dictionary.txt, extracts function opcode blocks, remaps 09 xx group index SFR references from another PCSX/function BIN when requested, patches/rebuilds sx_exe_image, and edits the packed permanent string table.

1) Load files

Use this when the loaded function .bin comes from a different PCSX. SFR references and permanent-string indices can be translated into the target PCSX.

2) Extract

Extracted opcode bytes are raw little-endian 16-bit VM words. buffer_len in the PCSX metadata is a word count; byte length is buffer_len * 2.

3) Reimport / add function

SFR bytes are little-endian VM words: 09 05 01 00 0A 00 means opcode 09 05, script_object-by-name group 1, function index 0x0A.

Advanced hash override
SFR opcode / permanent-string reference remap

Scanner recognizes these SFR opcode words: 09 2E, 09 05, 09 06, 09 07, 09 08, 09 09, 09 03, 09 04, 09 2D, 09 2C. The next word is the script_objects_by_name group and the next word is the function index in that group.

Manual mode rewrites exact SFR pairs. Example: source 1:0x0A retargets 09 05 01 00 0A 00 to the selected/reimported function. When adding a new function and manual source is blank, the tool first looks for 1:0x0A; otherwise it can retarget one unique SFR pair in the BIN.

Permanent string table

This is the packed permanent_string_table. Add new strings at the end to keep existing opcode string indices stable.
